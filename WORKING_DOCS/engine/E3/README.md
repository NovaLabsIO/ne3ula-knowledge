# E3

E3 is the ENGINE of NE3ULA.

If NE3ULA provides the WORLD of meaning, E3 provides the mechanics that move a
participant through transformation, reflection, alignment, and return.

E3 is not the myth itself.
It is the system that helps a person work with the myth in a repeatable way.

## Core Function

E3 exists to answer a practical question:

How does transformation happen inside NE3ULA?

Across the archive cluster, E3 consistently describes transformation through:

- authored identity
- reflective path systems
- behavioral calibration
- decision support
- adaptive feedback
- AI-assisted mirroring

## Internal Structure

E3 is organized into two canonical layers:

- `human-engine/`
- `system-engine/`

### Human Engine

The human engine contains the frameworks a participant directly uses to orient,
interpret, and reshape their life.

Its primary components are:

- Identity Stack
- Alchemist Pyramid
- Decision Engine

### System Engine

The system engine contains the technical, tracking, and adaptive layers that
support the human engine without replacing it.

Its primary components are:

- AI.d
- instrumentation

## Canonical Principles

The E3 archive cluster establishes several durable principles that define the
engine as a whole:

- transformation is cyclical, not strictly linear
- authored identity matters more than passive self-description
- reflection should become embodied action
- drift is normal and return is learnable
- systems should support sovereignty, not replace it
- AI assistance should mirror signal, not dictate identity

## Relationship to Other Layers

- NE3ULA provides meaning, story, and symbolic context
- E3 provides transformation mechanics
- AXZIO provides navigation and interaction
- DOMAINS express outcomes of transformation
- SYSTEMS enable participation and infrastructure

## Archive Synthesis Notes

The source cluster includes several adjacent but non-engine ideas, including:

- traveler entry and initiation material
- community identity naming
- future tech and artifact concepts
- symbolic music and interactive experience experiments

Those concepts informed this synthesis but were not forced into E3 canon when
they more clearly belonged to WORLD, SYSTEMS, DOMAINS, or INTERFACE.

## Meta-Development Pattern

One recurring internal pattern in the archive is the movement from lived
experience to repeatable module.

This appears most clearly in the Stretch Box material:

lived experience
-> distilled insight
-> structural pass through E3 frameworks
-> repeatable path / ritual / tool / module

That process is useful as an internal engine-development method, but it is not
itself the core ontology of E3.
