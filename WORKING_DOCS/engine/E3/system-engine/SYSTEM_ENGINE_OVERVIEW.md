# System Engine Overview

The system engine is the adaptive support layer of E3.

It translates human-engine activity into reflective feedback, pattern awareness,
and optional AI-supported mirroring.

## Subsystems

### AI.d

AI.d is the adaptive reflective core of the system engine.
It learns from accepted inputs, rituals, patterns, and participant responses in
order to mirror back useful observations.

### Instrumentation

Instrumentation is the broader tracking and feedback layer that captures rhythms
such as:

- ritual consistency
- mode distribution
- friction events
- drift signals
- archetype shifts
- alignment summaries

## Functional Boundary

The system engine should:

- support the human engine
- make patterns more visible
- help the participant notice return paths

The system engine should not:

- override the participant's authorship
- turn E3 into a surveillance model
- treat reflection as a scorecard
