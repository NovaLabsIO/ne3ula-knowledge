# System Engine

The system engine contains the technical and adaptive layers of E3.

Its job is to support the human engine through reflection, tracking, pattern
detection, and feedback loops.

It should never replace human authorship.

## Canonical Components

- `ai.d/`
- `instrumentation/`

## Core Function

The system engine exists to:

- observe patterns over time
- surface reflective prompts
- detect drift and recovery
- summarize alignment rhythms
- support return to signal

## Governing Rule

The archive is consistent on one point:

AI and instrumentation should mirror the participant's chosen signal.
They should not become external authority over identity.
