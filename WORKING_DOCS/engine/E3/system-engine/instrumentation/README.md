# Instrumentation

Instrumentation is the observation and feedback layer within the E3 system
engine.

It gathers meaningful patterns from human-engine activity so that reflection can
become more precise over time.

## Canonical Scope

The archive points to instrumentation around:

- ritual completion and collapse
- mode distribution and imbalance
- friction events
- drift detection
- path completions
- archetype-expression changes
- timeline markers and signal summaries

## Function

Instrumentation exists to make change visible.

Its purpose is not surveillance.
Its purpose is to answer questions such as:

- What rhythms am I actually living?
- Where does drift tend to begin?
- What helps me recover?
- Which patterns repeat across weeks or seasons?

## Main Output Types

The archive's most stable instrumentation outputs are:

- reports
- trend maps
- drift logs
- signal cards
- story exports

These outputs should be interpreted as reflective artifacts, not objective
judgments about worth or progress.

## Design Rule

Instrumentation should always remain subordinate to participant sovereignty and
the human engine's authored frameworks.

It is useful only when it helps a person notice signal, static, rhythm, and
return more clearly.
