# Human Engine

The human engine contains the participant-facing transformation frameworks inside
E3.

It is the part of the ENGINE that helps a person define who they are, recognize
how they operate, and move through structured growth without losing authorship.

## Canonical Components

- `identity-stack/`
- `alchemist-pyramid/`
- `decision-engine/`

## Function

The human engine exists to support:

- identity authorship
- path-based transformation
- reflection and integration
- rhythmic behavior change
- more conscious decision-making

## Design Logic

Across the archive cluster, the human engine is built on a simple pattern:

1. clarify signal
2. name current state
3. walk a path of reflection
4. translate insight into action
5. notice drift
6. return to alignment

This layer should always remain human-centered.
It is where E3 becomes lived practice rather than abstract architecture.
