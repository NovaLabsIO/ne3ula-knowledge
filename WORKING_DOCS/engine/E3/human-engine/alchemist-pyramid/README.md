# Alchemist Pyramid

The Alchemist Pyramid is the primary structured path framework inside the E3
human engine.

It organizes transformation as a layered system of orientation, force, practice,
integration, and return. In canonical terms, it is one of the main ENGINE
frameworks that translates NE3ULA's WORLD-level meaning into repeatable human
development mechanics.

This directory is therefore not a lore area and not a loose collection of
reflective essays. It is a path architecture.

## Why It Lives in `human-engine`

The Pyramid sits in `WORKING_DOCS/engine/E3/human-engine/` because it describes
how a participant moves through transformation.

Its role is to help a participant:

- locate their current orientation
- recognize the deeper forces shaping experience
- engage specific paths of practice
- interpret lived tension without collapsing complexity
- move toward more coherent integration

WORLD provides meaning, cosmology, and symbolic context.
The Alchemist Pyramid provides one of the core ENGINE structures for how that
meaning is metabolized in human experience.

## Canonical Pyramid Hierarchy

The reconstructed canonical hierarchy is:

Hermetic Base
-> foundational principles

Archetypes
-> identity-pattern layer

Core Energies
-> underlying forces of experience

Elements
-> operational paths of practice

Modes
-> practical life orientations

Polarity
-> reflective integration of opposing forces

Alchemist
-> final integrative layer

This ordering reflects the most coherent progression recoverable from the
archive while keeping the framework maintainable inside the NE3ULA ontology.

## Layer Distinctions

Three layers must remain clearly distinct:

- Archetypes are recurring identity-patterns or orientations of embodiment.
- Core Energies are underlying motivational or experiential forces.
- Elements are operational paths through which those forces are worked in
  practice.

These layers relate to one another, but they are not interchangeable.
Archetypes answer how a participant tends to engage the path.
Core Energies answer what deeper force is active.
Elements answer where and how that force is practiced, confronted, or refined.

## Folder Guide

### `hermetic-base/`

The foundational principle layer of the Pyramid.

This folder holds the governing assumptions and reality principles that inform
the rest of the system. It provides the interpretive base for causality,
rhythm, polarity, change, and pattern.

### `archetypes/`

The identity-pattern layer.

This folder defines recurring orientations such as Seeker, Builder,
Challenger, and Architect. These are not rigid personality labels. They are
entry-patterns and embodiment styles that shape how a participant tends to
approach transformation.

### `core-energies/`

The underlying force layer.

This folder captures the deeper energetic drivers named in the framework, such
as Freedom, Thought, Purpose, and Love. These are not paths and not roles.
They are the forces moving beneath behavior, motivation, and conflict.

### `elements/`

The operational path layer.

This folder holds the major practice paths, including Mirror, Shadow, Flame,
Frequency, and Vision. These are the structured routes through which insight is
tested, reflected, practiced, and integrated.

Elements are distinct from Core Energies.
Core Energies describe the force beneath movement.
Elements describe the path through which that movement is worked.

### `modes/`

The practical life-orientation layer.

This folder captures how a participant's current signal tends to express
itself in lived life, especially through orientations such as Pleasure,
People, and Production. Modes are situational and behavioral. They are not the
same thing as identity-patterns or energies.

### `polarity/`

The integration-through-opposition layer.

This folder contains the reflective material for tension, contradiction,
duality, and opposing truths. Its purpose is not to eliminate polarity, but to
help participants hold it consciously enough for integration to occur.

### `alchemist/`

The final integrative layer.

This folder represents the point at which prior layers converge. It is the
place where the participant returns with greater coherence, wider awareness,
and a more integrated relationship to force, path, and identity.

## Supporting Frameworks

### `MBHS/`

MBHS stands for Mind, Body, Heart, Spirit.

It is a supporting human integration framework used across the Pyramid. It is
not a formal tier of the Pyramid itself. Its purpose is to keep reflection and
practice from collapsing into cognition alone by preserving four major human
dimensions.

### `ring-framework/`

RING stands for Mythos, Logos, Pathos, Ethos.

It is a supporting interpretive lens used across Pyramid work. Like MBHS, it is
not a formal Pyramid tier. It provides a four-angle way of reading story,
logic, emotion, and values inside any layer or path.

## Canonical Docs vs `_source`

All folders in this directory except `_source/` are canonical ENGINE
documentation.

They are the place for:

- refined terminology
- ontology-aligned structure
- ongoing clarification
- cross-document consistency
- long-term maintenance

### `_source/`

`_source/` is a preserved 1:1 reference copy of the original archive subtree
used to reconstruct this framework.

It exists for:

- traceability
- fidelity checking
- recovery of detail not yet fully synthesized
- validation of canonical interpretations against source material

It is not the canonical destination for future Pyramid development.
It should not become the place where edits accumulate.

## Long-Term Maintenance Model

The long-term architecture of this framework depends on a clean separation
between canonical structure and preserved source material.

Canonical docs should carry the stable, readable, ontology-aligned version of
the Pyramid.

`_source/` should remain a historical and migration reference layer.

When revising this framework:

1. Start with the relevant canonical layer document.
2. Consult `_source/` only when clarification, verification, or missing detail
   is needed.
3. Update the canonical document, not the preserved source copy.
4. Preserve the distinctions between Archetypes, Core Energies, Elements, and
   Modes.
5. Prefer fewer, stronger canonical documents over archive-shaped duplication.

## Structural Intent

This directory should remain legible as a layered transformation system.

It should not drift into:

- flattened summary prose
- archive-first organization
- collapsed layers that blur identity-pattern, force, and practice

The canonical goal is clarity, fidelity, and maintainability without reducing
the framework to abstraction alone.
