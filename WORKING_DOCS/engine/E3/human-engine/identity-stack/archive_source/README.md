# Identity Stack Source Archive

This directory contains a selective preserved copy of the raw identity-stack
material imported from `NOTION_ARCHIVE/`.

Purpose
-------
The Identity Stack is the active migration target for the E3 human engine.
This `archive_source/` subtree preserves the smallest source cluster needed to
reconstruct the canonical identity documents while keeping the migration legible
and traceable.

The canonical working documents live in:

`WORKING_DOCS/engine/E3/human-engine/identity-stack/`

Those canonical documents provide:

- ontology-aligned structure
- refined terminology
- concise synthesis
- stable definitions for future AXZIO and decision-engine alignment

Rules
-----
- Treat this directory as preserved reference material.
- Do not treat these files as canonical.
- Do not edit raw wording except where filename or filesystem safety required it.
- Do not expand this folder into a general working area.

Selection Scope
---------------
This import is intentionally selective.

It preserves raw material related to:

- `Your Identity Stack`
- `Identity Core`
- `Identity Engine`
- identity signals and drift
- small AI.d reading materials directly relevant to identity interpretation

It does not preserve the broader AI.d architecture or unrelated decision
frameworks at this stage.

Migration Use
-------------
When revising the canonical identity stack:

1. Read the corresponding source file in `archive_source/`.
2. Extract the conceptual structure and identity logic.
3. Rewrite into the canonical document.
4. Keep canonical docs concise and ontology-aligned.
5. Preserve ambiguity explicitly when the archive is not yet settled.
