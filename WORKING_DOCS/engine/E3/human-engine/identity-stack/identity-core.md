# Identity Core

Identity Core defines the foundational identity primitives used by the E3 human
engine.

It is the authored center of the participant: the layer of orientation,
commitment, and self-definition that the rest of the identity stack reads
against.

## Purpose

Identity Core exists to distinguish authored identity from:

- inherited scripts
- situational moods
- social performance
- temporary behavioral states

The system needs a stable reference layer in order to interpret change.
Identity Core provides that reference layer.

## Foundational Structure

Identity Core is not a mood, role, or profile.
It is the deeper identity structure a participant is organizing life around.

This includes:

- authored identity
- core orientation
- values and commitments
- recurring pattern
- self-concept versus lived embodiment
- stable structure versus temporary state

These elements give the system a grounded way to interpret who the participant
is trying to be, how consistently that identity is being lived, and where drift
or fragmentation is occurring.

## Core Questions

Identity Core helps answer questions such as:

- What identity is this person consciously authoring?
- What principles are they actually organized around?
- What commitments remain stable across changing circumstances?
- What pattern keeps repeating beneath surface behavior?
- Where is there mismatch between declared identity and embodied life?

These questions sit beneath practical life navigation.
They concern identity structure, not just current conditions.

## Structural Reading

Identity Core should be read as the participant's relatively stable identity
architecture.

It changes more slowly than:

- mood
- current mode
- recent behavior
- stress response
- situational adaptation
- social labeling

A participant may act out of alignment with their core without the core itself
having changed.

The system should therefore treat Identity Core as a stable interpretive
reference while using signals, patterns, and readings to determine whether that
identity is being embodied, suppressed, revised, or distorted.

## Relationship to Applied Models

Some models inside the Identity Stack describe practical developmental domains
or applied interpretive lenses.

Those models are useful, but they are not the total definition of Identity
Core.

The Four Primitives model, for example, helps AXZIO interpret grounded
developmental concerns across money, engagement, building, and being.
It should be read as an applied model within the stack, not as the whole of the
participant's foundational identity structure.

## Canonical Principle

Identity in E3 is not primarily a static label.
It is a structured authorship layer that gives the system a coherent way to
recognize drift, coherence, and development over time.
