# Decision Engine

The decision engine contains the practical heuristics and tools inside E3 that
help a participant move from reflection into clear action.

It is not a single framework.
It is a cluster of decision-support mechanics drawn from the archive.

## Core Function

The decision engine helps answer:

- What matters now?
- What should I do next?
- What behavior supports my signal?
- What is friction, and what is growth?

## Canonical Components

### Prioritization

The archive repeatedly points toward focus-based prioritization tools, including
the One Thing concept and the Eisenhower Matrix.

These should be interpreted canonically as tools for reducing noise and choosing
the next meaningful action.

### Behavioral Loop Design

The Habit Engine material is sparse, but it clearly indicates that E3 values
repeatable behavior loops as mechanisms for turning insight into embodiment.

This belongs in the decision engine as action architecture rather than identity
definition.

### Activation and Confidence Support

The confidence formula material reframes confidence as an activatable state made
of curiosity, competence, congruence, and connection.

This fits the decision engine as a readiness model for action rather than as a
standalone identity doctrine.

### Systems Thinking

The MBHS logic branch and Architect material both imply that decision quality
improves when the participant can see patterns, tradeoffs, and feedback loops.

Systems thinking should therefore be treated as a decision-engine stance.

## Relationship to Other Parts of E3

- Identity Stack defines the authored center
- Alchemist Pyramid reveals path context and developmental terrain
- Decision Engine chooses actions inside that context
- System-engine instrumentation observes what happens next
