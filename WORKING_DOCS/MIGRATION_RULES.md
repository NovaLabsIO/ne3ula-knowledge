# Migration Rules

These rules govern how material moves from `NOTION_ARCHIVE` into `WORKING_DOCS`.

## Core Principle

`NOTION_ARCHIVE` preserves history.  
`WORKING_DOCS` creates clarity.

Do not move archive material directly without interpretation.

## Rules

1. Treat the archive as source material, not as a canonical structure.
2. Place refined material according to ontology, not legacy folder names.
3. Prefer synthesis over duplication.
4. Preserve legacy naming only when it is still conceptually necessary.
5. If a concept spans multiple layers, choose its primary home and reference the overlap in related files.
6. Do not allow implementation convenience to distort conceptual clarity.
7. When uncertain, document ambiguity explicitly rather than forcing a premature placement.

## Workflow

Archive item
→ interpret meaning
→ map to ontology
→ refine language
→ place in WORKING_DOCS
→ later connect to implementation repos
